WebSocket API

Real-time bidirectional communication using Socket.IO

WS ws://localhost:3000

Connect to the WebSocket gateway for real-time messaging

Parameter Type Required Description
username string Optional Username for the connection
token string Optional JWT token for authenticated messages
JavaScript Connection Example 
const socket = io('http://localhost:3000', {
  auth: {
    username: 'JohnDoe'
  }
});

socket.on('connect', () => {
  console.log('Connected!', socket.id);
});

socket.on('welcome', (data) => {
  console.log(data.message);
});
EVENT message

Send a broadcast message to all connected clients

Example 
// Send message
socket.emit('message', {
  message: 'Hello everyone!'
});

// Receive messages
socket.on('message', (data) => {
  console.log(`${data.username}: ${data.message}`);
});
EVENT joinRoom

Join a specific room for targeted messaging

Parameter Type Required Description
room string Required Name of the room to join
Example 
// Join room
socket.emit('joinRoom', { room: 'developers' }, (response) => {
  console.log('Joined room:', response.room);
  console.log('Members:', response.members);
});

// Listen for room events
socket.on('userJoinedRoom', (data) => {
  console.log(`${data.username} joined ${data.room}`);
});
EVENT roomMessage

Send message to a specific room only

Example 
// Send to room
socket.emit('roomMessage', {
  room: 'developers',
  message: 'Hello developers!'
});

// Receive room messages
socket.on('roomMessage', (data) => {
  console.log(`[${data.room}] ${data.username}: ${data.message}`);
});
EVENT privateMessage

Send a private message to a specific user

Parameter Type Required Description
targetId string Required Socket ID of the recipient
message string Required Message content
Example 
// Send private message
socket.emit('privateMessage', {
  targetId: 'user-socket-id',
  message: 'Secret message!'
});

// Receive private messages
socket.on('privateMessage', (data) => {
  console.log(`Private from ${data.fromUsername}: ${data.message}`);
});

Redis Pub/Sub API

Publisher/Subscriber pattern for horizontal scaling

GET /redis-pubsub

Get pattern information and examples

curl 
curl http://localhost:3000/redis-pubsub
POST /redis-pubsub/publish

Publish a message to a Redis channel

Parameter Type Required Description
channel string Required Channel name to publish to
message string Required Message content
curl 
curl -X POST http://localhost:3000/redis-pubsub/publish \
  -H "Content-Type: application/json" \
  -d '{
    "channel": "news",
    "message": "Breaking news!"
  }'
JSON Response 
{
  "success": true,
  "channel": "news",
  "message": "Breaking news!",
  "subscriberCount": 2,
  "timestamp": "2025-01-15T10:30:00.000Z"
}
POST /redis-pubsub/subscribe

Subscribe to a Redis channel

curl 
curl -X POST http://localhost:3000/redis-pubsub/subscribe \
  -H "Content-Type: application/json" \
  -d '{"channel": "news"}'
GET /redis-pubsub/stream?channel=news

Server-Sent Events (SSE) stream for real-time updates

JavaScript Example 
const eventSource = new EventSource(
  'http://localhost:3000/redis-pubsub/stream?channel=news'
);

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(`${data.channel}: ${data.message}`);
};

NATS RPC API

Request/Response pattern for microservices communication

POST /nats-rpc/request

Send an RPC request and wait for response

Parameter Type Required Description
subject string Required NATS subject name
data string Required Request payload
curl 
curl -X POST http://localhost:3000/nats-rpc/request \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "greet",
    "data": "World"
  }'
POST /nats-rpc/publish

Publish a fire-and-forget message to a subject

curl 
curl -X POST http://localhost:3000/nats-rpc/publish \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "events",
    "data": "User logged in"
  }'

RabbitMQ API

Background job processing with reliable message queues

POST /rabbitmq/send

Send a job to the message queue

Parameter Type Required Description
queue string Required Queue name
message string Required Job payload
curl 
curl -X POST http://localhost:3000/rabbitmq/send \
  -H "Content-Type: application/json" \
  -d '{
    "queue": "tasks",
    "message": "Process video upload"
  }'
GET /rabbitmq/status

Check RabbitMQ connection status

curl 
curl http://localhost:3000/rabbitmq/status

Authentication

WebSocket Authentication

For authenticated WebSocket messages, provide a JWT token during connection:

Example 
const socket = io('http://localhost:3000', {
  auth: {
    token: 'your-jwt-token-here',
    username: 'JohnDoe'
  }
});

// Send authenticated message
socket.emit('authenticatedMessage', {
  message: 'This requires authentication'
});

HTTP API Authentication

For authenticated HTTP endpoints, include the Authorization header:

Example 
curl -X POST http://localhost:3000/endpoint \
  -H "Authorization: Bearer your-jwt-token" \
  -H "Content-Type: application/json" \
  -d '{"data": "value"}'
